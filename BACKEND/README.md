# BACKEND
🌟 VibeCheck — Backend API
Это серверная часть приложения для отслеживания настроения, разработанная на языке Go. Проект настроен для интеграции с базой данных MySQL и предоставляет API-эндпоинты для фронтенд-разработки.

📂 Структура проекта
Проект организован по принципу разделения ответственности, что облегчает работу в команде.
/backend
├── cmd
│   └── server
│       └── main.go        // Точка входа: запуск сервера и подключение к БД
├── internal
│   ├── handlers
│   │   └── handlers.go    // Логика обработки запросов и ответы API (201, 400, 500)
│   ├── models
│   │   └── models.go      // Описание структур данных (User, MoodLog)
│   └── repository
│       └── database.go    // SQL-запросы: взаимодействие с таблицами MySQL
├── .env                   // Настройки окружения (порты, пароли БД)
├── go.mod                 // Зависимости проекта
└── go.sum                 // Контрольные суммы библиотек

Подробное описание компонентов:
cmd/server/main.go: Основной файл для запуска. Он инициализирует соединение с базой данных через порт 3307 и запускает сервер на порту 8080.

internal/handlers/: Содержит функции, которые принимают данные от пользователя. Например, обработчик регистрации проверяет входящий JSON и возвращает статус 201 Created при успехе.

internal/models/: Определяет "скелет" данных. Здесь прописано, какие поля должны быть у пользователя (username, password) и у записей настроения.

internal/repository/: Единственное место, где пишется чистый SQL-код (INSERT, SELECT). Это позволяет менять логику хранения данных, не затрагивая остальную часть программы.

🛠 Настройка и запуск
База данных:

Убедитесь, что сервер MySQL запущен на порту 3307.

Используйте файл vibe_check.sql для создания структуры таблиц (users, mood_logs, tags).

Переменные окружения:

В файле .env укажите параметры подключения к вашей локальной базе данных.

Запуск сервера:
cd BACKEND
go run cmd/server/main.go

📡 Документация API (Эндпоинты)
1. Регистрация пользователя
URL: http://localhost:8080/api/register

Метод: POST

Body (JSON):
{
  "username": "aelita_student",
  "password": "password123"
}

Результат: 201 Created (данные успешно сохранены в таблицу users).

2. Сохранение лога настроения
URL: http://localhost:8080/api/logs/save

Метод: POST

Результат: Запись появляется в таблице mood_logs в MySQL Workbench.
